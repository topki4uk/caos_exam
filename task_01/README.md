## Стадии сборки

### Всего есть 4 стадии сборки:

1) Препроцессинг
    ```bash
   g++ -E example.cpp -o advanced.cpp
   ```
   
2) Компиляция
   ```bash
   g++ -S example.cpp -o compiled.asm
   ```

3) Ассемблирование
   ```bash
   g++ -c example.cpp -o example.o
   ```
   
   Чтобы посмотреть внутренности `*.o` файла, необходимо использовать команды для просмотра бинарных файлов,
   например, программой `objdump` или `hexdump`.
   ```bash
   objdump -D example.o
   ```

4) Линковка - это процесс, при котором файл связывается с другими `*.o` файлами и дополняется необходимыми данными, 
   чтобы можно было его запускать
   ```bash
   g++ example.o
   ./a.out
   ```
   
### Чем объектный файл отличается от исполняемого?
Объектный файл это незаконченный продукт, он не пригоден для запуска, потому что не содержит внутри себя компонент,
которые позволили бы ему корректно отрабатывать. Именно на этапе линковки файл дополняется необходимыми ссылками и адресами,
что, например, позволяет ему обращаться к другим библиотекам или использовать системные вызовы.

### Как дизассемблировать исполняемый файл?
Для дизасемблирования файла можно воспользоваться специальными утилитами, которые возвращают нам информацию о 
файле, например, `hexdump`, `objdump` и `readelf`.

```bash
  # Для hexdump
  hexdump -C a.out
```

```bash
  # Для objdump
  objdump -D a.out
```

```bash
  # Для readelf
  readelf -a a.out
```
