## Какие версии сисколла exec существуют и в чем разница между ними?

| Функция   | Аргументы | PATH поиск | Переменные окружения |
|-----------|-----------|-----------|-----------------------|
| `execl`   | Список    | Нет       | Наследует             |
| `execlp`  | Список    | Да        | Наследует             |
| `execle`  | Список    | Нет       | Передать отдельно     | 
| `execv`   | Массив    | Нет       | Наследует             | 
| `execvp`  | Массив    | Да        | Наследует             | 
| `execvpe` | Массив    | Да        | Передать отдельно     | 
| `execve`  | Массив    | Нет       | Передать отдельно     | 

## В чем необычность функций `fork` и `exec`, что происходит при их вызове?

Несмотря на то что мы вызываем их как функции в программе, они не работают как функции.

* `fork` будет порождать "мультивселенные", у нас образуется процесс-ребенок, который будет выполнять свои инструкции.
* `execve` полностью подменяет программу, обнуляется все: стек, куча, выделенная память, но `PID` остается.

## Что такое fork-бомба?

`Fork-бомба` - это вид DoS-атаки (Denial of Service) на систему, при которой процесс рекурсивно создает свои копии,
пока не исчерпаются системные ресурсы.

```text
# Самая известная fork-бомба
:(){ :|:& };:
```

```c++
int main() {
    while(1) {
        fork();  // Бесконечное создание процессов
    }
    return 0;
}
```
