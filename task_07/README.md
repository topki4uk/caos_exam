# Что такое сборка в дебаг-режиме и в чем ее отличие от обычной?

Для запуска в дебаг-режиме нужно компилировать с флагом `-g`:

```shell
  g++ -g example.cpp -o debug_example
```

# Как с помощью gdb отлаживать программу?

* `gdb ./<filename>` - запуск программы в дебаг режиме
* `run` - начало работы
* `exit` - выход из программы
* `p <var>` - вывод переменной
* `b <num>` - поставить брейкпоинт
* `bt` - посмотреть бэктрейс
* `n` - шаг без захода в функцию
* `s` - шаг с заходом в функцию

# Что означает фраза “core dumped” и как с помощью gdb посмотреть содержимое coredump-файла после того, как программа упала?

Файлы сохраняются по адресу: `/var/lib/systemd/coredump`
