## Что такое istruction-level parallelism в процессоре?

Instruction-Level Parallelism (ILP, Параллелизм на уровне инструкций) — это способность процессора одновременно исполнять несколько машинных инструкций за один такт, 
даже если они идут последовательно в программе. Это не про многопоточность (thread-level), а про извлечение параллелизма из одного потока инструкций.

## Что такое `out-of-order` execution в процессоре, к каким неожиданным эффектам это может приводить?

- Современные процессоры могут переупорядочивать инструкции для лучшего использования ресурсов.
- Условие: отсутствие зависимостей по данным
- Результат сохраняется в порядке программы благодаря `reorder buffer`

## Что такое спекулятивное исполнение кода в процессоре?

Процессор, как уже выяснилось, старается всегда выполнять инструкции наперед.
С тем же Branch Prediction'ом, когда до конца не известно, надо ли будет выполнить инструкцию или нет, процессор уже может начать ее выполнять. 

Это называется `Speculative execution`

```c++
if (num != 0 && 10 / num == 3) {
	// ...
}
```

## Расскажите в общих чертах, к каким уязвимостям оно может приводить.

#### Уязвимость Spectre

План: тренировать Branch Prediction на if'е, который всегда срабатывает на `true`. 
Затем заставляем процессор обратиться к памяти спекулятивно (процессор будет выполнять, 
потому что уже натренирован на `true`). 
Он поймет, что "обжегся", заготовит `SegFault`, но выдавать его не будет. 
Вдруг окажется, что Branch Predictor ошибся, поэтому SegFault не вызовется.
